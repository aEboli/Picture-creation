## 当前技术结论

| 领域 | 当前采用方案 | 状态 | 备注 |
| --- | --- | --- | --- |
| Web 框架 | `Next.js App Router + React + TypeScript` | Confirmed | 页面与 API route 同仓库、同构建 |

| 生成主路径 | 后端自动化 quick 流程 + Prompt 质量增强 + 视觉质检 | Confirmed | 当前不再以前端策略工作台为主入口 |
| 创建页辅助 | `/api/agent-chat` + 本地智能体侧窗 | Draft | 当前已接入创建页，但尚未形成独立发布文档口径 |
| 模板能力 | 保留数据库/历史兼容，页面与 API 入口退役 | Confirmed | 避免继续扩散旧入口 |

## 第三方服务与关键依赖

| 名称 | 用途 | 状态 | 风险/约束 |
| --- | --- | --- | --- |
| `@google/genai` | Gemini 文本与图像模型调用 | Confirmed | 依赖用户配置 API Key、模型名、可选 Relay 参数 |
| Gemini / Relay | 实际生成与测试链路 | Confirmed | 文本/图像模型配置错误会直接影响创建与智能体能力 |
| 飞书开放平台 | 同步任务与产出到 Bitable | Confirmed | 字段映射、权限、网络和配置完整度都会影响成功率 |
| `sharp` | 资产缩略图与图片处理 | Confirmed | Windows 发布包必须带齐原生运行时文件 |
| SQLite | 本地持久化任务、设置、品牌等数据 | Confirmed | 无用户级鉴权，默认依赖本地使用场景 |

## 实际落地偏差

- 原计划：`v0.7.0` 对外文档仍以“模板中心可用、五模式完整交付、品牌更名完成”为主结论
- 实际采用：
  - 模板中心页面与 API 已退役
  - 创建主路径默认收口到后端自动化 quick 流程
  - 创建页额外挂了智能体侧窗，用于表单补全与 Prompt 建议
- 偏差原因：后续迭代先落代码和局部文档，主技术结论没有同步回填
