# 版本说明-v0.9.0

对应对象：`Picture-creation`

发布日期：2026-05-05

## 本版重点

- 创建页生成流程、创建页智能体和服务测试统一复用浏览器本地 API 配置。
- 设置页新增图型提示词编辑能力，覆盖标准模式、套图模式和亚马逊 A+ 模式。
- 图型提示词进入 Gemini 与 OpenAI 两条 provider 的结构化生成链路。
- 侧栏 `API` 与 `飞书` 状态胶囊可以直接打开对应快捷配置抽屉。
- 创作台输出参数区恢复到同一工作台表面，比例、分辨率和图型选择保持主流程内可见。
- 默认市场参数固定为美国、英文、Amazon，避免中文界面误改生成市场。

## 核心更新

1. `package.json` 与 `package-lock.json` 版本提升到 `0.9.0`，安装器版本名同步为 `PICTURE-CREATION-WINDOWS-0.9.0.exe`。
2. 创建页提交生成时读取浏览器本地 API 设置，并通过 `temporaryProvider` 传给后端：
   - `provider`
   - `apiKey`
   - `apiBaseUrl`
   - `apiHeaders`
   - `textModel`
   - `imageModel`
3. 创建页智能体复用浏览器本地 API Key 和文本模型；OpenAI 兼容 provider 走 Responses API，并使用文本模型字段。
4. API 连接测试成功后会同步写入浏览器本地 API 设置，减少“测试成功但生成仍未配置”的断层。
5. 设置页新增 `生图类型提示词` 折叠面板，支持按模式和图型编辑提示词规则。
6. 新增 `image_type_prompt_overrides_json` 设置字段，数据库、默认设置、服务端校验与保存逻辑同步支持。
7. Gemini 与 OpenAI 的结构化 copy/prompt builder 均接收解析后的图型提示词。
8. 侧栏 API / 飞书状态胶囊变成快捷配置入口，可打开只包含对应配置区块的服务抽屉。
9. 创作台布局回归同一工作台表面，参考图模式下原图区与参考图区保持等高且不溢出。
10. 回归测试补充覆盖浏览器 API 生成传参、智能体 temporary provider、图型提示词设置、快捷配置抽屉和默认市场参数。

## 升级说明

- 默认数据目录继续使用 `%LOCALAPPDATA%\Picture-creation\data`。
- 如本地已有旧目录或旧数据库名，程序会自动兼容。
- 浏览器本地 API 设置需要使用者在自己的浏览器里填写，不随源码或发布包公开携带。
- 旧环境变量 `COMMERCE_STUDIO_*` 仍保留兼容，便于已有脚本过渡。

## 验收关注点

- 构建前若存在运行中的 `.next/standalone/server.js`，需要先释放占用。
- 打包后需检查桌面快捷方式名称是否为 `Picture-creation`。
- 需验证 API 测试成功后，创建页生成和创建页智能体能复用浏览器 API 设置。
- 需验证图型提示词折叠面板可保存，并且标准模式、套图模式和亚马逊 A+ 模式均能读取对应提示词。
- 建议在 GitHub 发布前再次确认 `.env`、`data/`、`release/`、运行截图、调试日志与 `.learnings/` 未被纳入源码提交。
