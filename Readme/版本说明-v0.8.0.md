# 版本说明-v0.8.0

对应对象：`Picture-creation`

发布日期：2026-03-30

## 本版重点

- 对外 README、使用说明、PRD 与版本说明统一切换到 `v0.8.0`。
- 品牌库现在是独立一级入口，设置页收口为紧凑控制中心。
- 设置页统一采用 `全局连接测试` 与 `保存全部设置` 的交互口径。
- 创建页离开不再打断式确认；详情页 copy 摘要与 `reference-remix` 缺省文案更清晰。
- 本次发布按公开 GitHub 仓库安全口径整理，默认排除本地诊断与运行产物。

## 核心更新

1. `package.json` 与 `package-lock.json` 版本提升到 `0.8.0`，安装器版本名同步为 `PICTURE-CREATION-WINDOWS-0.8.0.exe`。
2. `README.md`、`Readme/使用说明-Picture-creation.md`、`Readme/PRD-Picture-creation.md` 已改写为当前真实产品状态：
   - 主导航为 `总览 / 创作台 / 历史记录 / 设置 / 品牌库`
   - 品牌库独立成页
   - 模板中心维持退役
3. 设置页当前正式口径为：
   - 紧凑控制中心
   - Hero + L 型三卡布局
   - 底部统一状态区
   - `全局连接测试` + `保存全部设置`
4. 任务细节体验继续收口：
   - 创建页离开不再弹出草稿中断确认
   - 详情页 copy 摘要面板主标题改为顺序编号
   - `reference-remix` 缺少分析结果时，标题统一回落为 `Reference remake`
5. 发布前相关源码回归与类型检查已补齐，便于继续做 GitHub tag / Release 收口。

## 升级说明

- 默认数据目录继续使用 `%LOCALAPPDATA%\Picture-creation\data`。
- 如本地已有旧目录或旧数据库名，程序会自动兼容。
- 旧环境变量 `COMMERCE_STUDIO_*` 仍保留兼容，便于已有脚本过渡。
- 如果公开分发，建议优先使用脱敏发布产物，而不是直接上传本地开发运行目录。

## 验收关注点

- 构建前若存在运行中的 `.next/standalone/server.js`，需要先释放占用。
- 打包后需检查桌面快捷方式名称是否为 `Picture-creation`。
- 需至少验证设置中心、品牌库、历史记录和五种创建模式的主流程。
- 建议在 GitHub 发布前再次确认 `.env`、`data/`、`release/` 与 `.learnings/` 未被纳入源码提交。
