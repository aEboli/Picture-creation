# 使用说明-多图联合生成语义
对应对象：Picture-creation 创建页多图输入与请求语义

## 1. 适用范围

本说明适用于创建页的五种模式：

- 标准出图
- 套图模式
- 亚马逊 A+ 图模式
- 提示词模式
- 参考图复刻

## 2. 生成方式

创建页左侧“图片原图”卡片头部新增“生成方式”切换器，包含两种语义：

- `多图联合`
  - 多张原图作为同一次 Gemini 请求的平级输入。
  - 默认 `1 组输入 = 1 组请求`。
  - 右侧“实际生成数量”按 `输入组 × 类型 × 比例 × 分辨率 × 套图数量` 计算。
- `批量套模板`
  - 多张原图继续按历史行为逐张拆分为独立请求。
  - 右侧“实际生成数量”按 `原图张数 × 类型 × 比例 × 分辨率 × 套图数量` 计算。

当前默认值为 `多图联合`。

## 3. Gemini 3.1 上限解释

- 默认图片模型为 `gemini-3.1-flash-image-preview`。
- 官方的 `14 张图` 指的是“单次请求总输入上限”，不是“整个任务总上传数上限”。
- 因此：
  - `多图联合` 需要受 `14 张/次` 约束。
  - `批量套模板` 只要每次拆分后的单次请求不超限，整批上传多少张原图都可以。

## 4. 各模式行为

### 标准出图 / 套图模式 / 亚马逊 A+ 图模式

- `多图联合`：原图区多张图共同参与同一次生成。
- `批量套模板`：原图区多张图逐张跑同一套参数。

### 提示词模式

- 继续支持 `0 张图 + 纯提示词`。
- 上传 1 张或多张原图后：
  - `多图联合`：多张图与提示词一起参与同一次生成。
  - `批量套模板`：每张图分别套用同一提示词独立生成。

### 参考图复刻

- 保持“原图区 + 参考图区”双区结构。
- `多图联合` 只作用于原图区：
  - 原图区多张图作为一个联合输入组。
  - 参考图区继续作为参考图集合。
  - 两区合计受 `14 张/次` 总上限约束。
- `批量套模板`：
  - 原图区多张图逐张拆分请求。
  - 每次请求带 `1 张原图 + 全部参考图`。

## 5. 上传与计数规则

- 原图与参考图预览计数统一显示为 `X/X`。
- 提交区“实际生成数量”会随“生成方式”实时变化。
- 当当前选择会让单次请求超过模型上限时：
  - 提交会被阻断。
  - 创建页会提示当前选择超出 `14 张/次` 限制。
- 在参考图复刻中：
  - `多图联合` 下，参考图区剩余名额由当前原图区张数决定。
  - `批量套模板` 下，参考图区仍按“预留 1 张原图位”计算可上传上限。

## 6. 维护提示

- 新字段：`generationSemantics`
  - 可选值：`joint`、`batch`
  - 已打通表单 payload、任务持久化、任务构建和执行链路。
- 旧任务兼容：
  - 历史任务默认按 `batch` 解释，避免重试旧任务时行为漂移。
