# 版本说明-v0.7.0

对应对象：`Picture-creation`

发布日期：2026-03-17

## 本版重点

- 项目统一更名为 `Picture-creation`。
- 默认模型切换为 Gemini 3.1 组合。
- 五种模式统一支持 `多图联合 / 批量套模板`。
- 发布脚本、安装器与绿色包文件名全面切到 `picture-creation` 口径。
- README、使用说明、PRD 与截图资产完成重写整理。

## 核心更新

1. `package.json`、应用标题、默认数据目录、安装器名与 GitHub 仓库名统一为 `Picture-creation`。
2. `lib/server-config.ts` 新增新旧数据目录兼容策略，支持从 `Commerce-Image-Studio` 平滑迁移。
3. `scripts/package-release.ps1`、`scripts/build-installer.ps1`、`scripts/build-inno-installer.ps1` 全部切换到 `picture-creation` 系列产物名。
4. README 与 `Readme/` 交付文档按 `v0.7.0` 重写，并纳入最新界面截图资产。
5. 保持既有多图联合生成语义与 Gemini 3.1 图片上限逻辑不回退。

## 升级说明

- 新默认数据目录为 `%LOCALAPPDATA%\Picture-creation\data`。
- 如本地已有旧目录或旧数据库名，程序会自动兼容。
- 旧环境变量 `COMMERCE_STUDIO_*` 仍保留兼容，便于已有脚本过渡。

## 验收关注点

- 构建前若存在运行中的 `.next/standalone/server.js`，需要先释放占用。
- 打包后需检查桌面快捷方式名称是否为 `Picture-creation`。
- 需实测五种模式至少各一次，确保更名未影响创建流程。
