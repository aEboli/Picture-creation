## 目标用户

- 主要用户：电商运营、设计支持、商品团队
- 次要用户：本地部署维护者、需要打包交付的协作者
- 使用场景：在局域网或单机环境里，围绕商品原图、提示词、参考图、审核导出和飞书同步完成一条本地化 AI 生图工作流

## 核心目标

- 这个项目/功能要解决什么问题：
  - 把电商素材生成、任务排队、审核导出、品牌约束、模型配置和本地交付收进一套可运行的桌面化工作流。
  - 让五种创建模式共享同一套输入组织、历史追踪和调试出口，而不是散在脚本和临时 Prompt 里。
- 为什么现在要做：
  - `v0.7.0` 的外部文档已经完成品牌化交付，但当前本地工作区继续沿着“精简界面、强化创建链路、退役旧入口、补 AI 助手与质检骨架”的方向演进，缺一份能反映当前真实状态的主文档。

## 用户流程

1. 用户从哪里进入：从总览页进入 `创作台 / 历史记录 / 设置`，其中创建任务主入口是 `/create`。
2. 用户做什么操作：在创建页选择模式、上传原图或参考图、填写提示词与商品信息；必要时使用创建页智能体补全商品信息或优化提示词。
3. 系统给出什么反馈：任务创建后跳转到 `/jobs/{id}`，随后可在详情页或历史页查看状态、错误、审核结果与导出入口。
4. 用户如何完成目标：任务完成后在详情页审核单图，下载通过图，或手动触发飞书同步与任务重试。

## 关键用户旅程

| 旅程 | 起点 | 关键步骤 | 成功结果 |
| --- | --- | --- | --- |
| 标准生图 | `/create` | 上传原图 -> 选择模式/图型/比例/分辨率 -> 提交 | 成功创建任务并进入详情页查看结果 |
| Prompt 创作 | `/create` | 输入 `promptInputs[]` -> 可不上传图片 -> 提交 | 每条提示词生成独立结果，详情页能看到最终发送 Prompt |
| 审核导出 | `/history` | 打开任务详情 -> 审核图片 -> 下载通过图 | 导出包可直接交付 |
| 参数补全 | `/create` | 打开智能体侧窗 -> 上传图/输入文本 -> 映射到表单 | 商品名称、卖点、材质、尺寸、品牌等字段被快速回填 |

## 完成标准

| 功能/流程 | 完成标准 | 失败提示/异常处理 |
| --- | --- | --- |
| 五种模式创建 | `POST /api/generate` 能创建并入队合法任务 | 非法输入返回明确错误，如数量超限、图片数量不符、`0.5K` 已退役 |
| Prompt 模式 | `promptInputs[]` 生效，旧 `customPrompt` 能兼容迁移 | 无有效提示词时返回 `Prompt mode requires at least one text prompt input.` |
| 生成执行 | 详情页能看到任务与单图状态、错误和最终 `promptText` | `provider-request / response / provider-image-download / visual-audit` 等阶段可定位 |
| 审核与交付 | 详情页能审核、重试、下载通过图、触发飞书同步 | 接口失败返回可读错误并保留原任务详情 |
| 创建页智能体 | 能返回 `assistantText + fieldMapping`，并只映射表单字段 | 未配置文本模型/API Key、非法 `agentType`、非法历史 JSON 时给出明确错误 |
| 旧入口退役 | 模板页不再作为可见页面入口，模板 API 返回 `410` | 访问退役接口时返回统一 retired 提示 |

## 页面/导航摘要

- 视觉风格：当前采用对称、液态玻璃感的工作台风格，顶部统一展示任务摘要与系统状态。
- 页面框架：总览页负责入口分发；创建页聚焦单一表单与智能体侧窗；历史页聚焦筛选和任务卡；详情页承接审核与排障；设置页承接配置与品牌库。
- 导航/信息架构：主导航只保留 `总览 / 创作台 / 历史记录 / 设置`，任务详情被归类到历史域；模板中心不再处于主导航。
- 关键页面/关键状态：
  - `/create`：支持模式切换、上传、多重提示词、智能体映射、提交后跳详情页。
  - `/history`：支持筛选、分页、任务卡和跳详情。
  - `/jobs/[id]`：支持轮询刷新、重试、飞书同步、审核、导出。
  - `/templates`：当前为退役页。

## 异常流与失败提示

| 场景 | 触发条件 | 用户提示 | 兜底处理 |
| --- | --- | --- | --- |
| 设置未配置 | 缺少 Gemini API Key 或文本模型 | 设置测试失败或接口返回配置缺失错误 | 引导用户先去设置页完成配置 |
| 创建参数非法 | 模式、图片数、分辨率或提示词不满足约束 | 接口返回明确错误文案 | 阻止任务入队，保留当前表单 |
| 模型返回非图片 | Provider 将请求当成文本任务 | 详情页 `providerDebug.failureStage = response` | 保留详情排障信息并允许重试 |
| 视觉质检失败 | 生成结果被 `visual-audit` 拦截 | 详情页可见失败原因 | 对单图位自动重生 1 次后再失败落库 |
| 智能体调用失败 | 非法历史 JSON、未配置设置、模型返回无效 JSON | 侧窗显示错误文案 | 不影响创建页主表单状态 |
| 访问退役入口 | 打开模板 API 或旧策略预览 API | `Template center has been retired.` 等 retired 文案 | 统一返回 404/410，避免继续扩散使用 |
